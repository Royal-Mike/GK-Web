group 4 - JWT
log in - authenticate user - create session token - check authorization
mỗi request phải query database để validate và kiểm ra quyền -> chậm
mở rộng database ? tốn chi phí -> JWT : token chứa cả thông tin về session và user
header - thoong báo
payload - chứa claim + metadata
signature - ký để bảo mật
client --log in--> Authentication Server --OK + JWT--> client --Request + JWT--> app server --OK--> client (quá trình verify JWT)
encode mà không encrypt
dễ bị lấy cắp
token đôi khi quá dài
expiration time đáng quan ngại
